# Proposal: 个人报销文件管理 App

## Intent
开发一个个人报销文件管理 App，解决消费后**结账单、支付记录、发票**三个文件的收集、关联、归档和月度打包发送问题。全程自动化管理，减少人工整理的工作量。

## Scope

### In Scope
- 拍照上传结账单并 OCR 识别，自动创建消费记录
- 每条消费记录关联 3 个文件：结账单照片、支付记录截图、发票 PDF
- 每日定时检查：记录缺支付记录 → 通知用户上传
- 每日定时检查邮箱：查找并下载关联的发票
- 月度自动打包：下月初将所有发票文件夹压缩，发送到固定邮箱
- 本月消费总额计算与展示

### Out of Scope (v1)
- 多人协作/审批流
- 自动报销提交到财务系统
- 多币种支持
- 发票真伪查验

## Approach
采用 **Flutter (前端) + Python FastAPI (后端)** 架构：
- 移动端使用 Flutter，一次开发同时支持 Android 和鸿蒙（通过 flutter_ohos）
- 后端 Python 处理 OCR、邮箱检查、定时任务和文件打包
- 本地数据库 SQLite，文件存储本地 + 可配置云备份
- 后端定时任务用 APScheduler 驱动每日检查和月初打包

## Key Design Decisions
1. Flutter 作为前端 — AI 友好的声明式 UI 框架，单代码库覆盖 Android + 鸿蒙
2. OCR 使用 PaddleOCR — 对中文小票识别率好，支持拍照识别
3. 邮箱使用 IMAP 协议 — 支持主流邮箱（QQ邮箱/163/Outlook等）
4. 文件按 `YYYY-MM/消费日期_摘要/` 层级组织
